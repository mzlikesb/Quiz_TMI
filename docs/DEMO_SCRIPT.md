# docs/DEMO_SCRIPT.md

## 데모 목표(4분)
- “음성 챗봇 데모”가 아니라 **barge-in이 룰인 게임**임을 보여준다.
- 점수/리더보드가 Firestore에 저장되는 **프로젝트성**을 증명한다.
- Cloud Run/Logging으로 **GCP에서 동작**함을 증빙한다.

---

## 0:00–0:20 오프닝(설명)
**내레이션(너):**
“이건 The Interruption Quiz입니다. 루카가 문제를 내지만 일부러 뇌절을 합니다.
사용자는 기다리면 손해고, **말로 끊어서(barge-in) 정답을 외쳐야 점수를 얻습니다.**”

화면: Unity 앱 메인(상태 표시 + Start 버튼 + 점수)

---

## 0:20–1:10 라운드 1 (barge-in 1회 + 채점)
1) Start
2) 루카가 문제 시작 + TMI로 떠듦 (일부러 길게)
3) **중간에 끊기**:
   - 너: “B!”
   - (만약 음성 인식 삑나면 자연스럽게: "어? 내 발음이 구린가? 그럼 이 버튼으로!" 하며 화면의 B 버튼 클릭. 실패/fallback 시연으로 오히려 신뢰감 어필)
4) 화면에서 즉시:
   - 루카 음성 **뚝 끊김**
   - 상태: `Interrupted → Judging → Score`
5) 루카 리액션:
   - “악! 내 말 안 끝났는데… 정답은 B!”
6) 점수 애니메이션(예: +145)

---

## 1:10–2:00 라운드 2 (barge-in 2회 + 속도 보상 강조)
1) 루카가 더 길게 뇌절 시작
2) **더 빨리 끊기**:
   - 너: “A!”
3) 점수가 더 크게 나오게 연출
4) 리더보드/누적 점수 업데이트

**내레이션(너):**
“정답뿐 아니라, 끊는 타이밍이 빠를수록 보상이 커집니다.”

---

## 2:00–2:40 ‘프로젝트’ 증명(저장/리더보드)
화면: Firestore 콘솔(또는 디버그 패널)
- `scores/{userId}` 문서에 누적 점수 업데이트 확인
- (선택) `runs/{runId}`에 이벤트 로그 저장 확인

**내레이션(너):**
“점수는 Firestore에 저장되어 리더보드로 바로 집계됩니다.
여기서 ‘음성 대화’가 아니라 게임 시스템으로 동작하는 걸 증명합니다.”

---

## 2:40–3:20 운영/안정성(재연결 또는 로그)
선택 A(권장): Unity에 `Simulate Drop` 버튼
- 눌러서 `Reconnecting…` 표시 → 자동 복구

선택 B: Cloud Run 로그 화면으로 이벤트 라인 보여주기
- `question_start`, `barge_in`, `judge`, `score_saved`

**내레이션(너):**
“실시간 앱은 끊김/복구가 중요합니다. 상태를 명확히 보여주고 자동 복구합니다.”

---

## 3:20–4:00 엔딩(요약)
**내레이션(너):**
“정리하면, 이 프로젝트는 Gemini Live의 저지연 음성을 ‘대화’가 아니라
**대화 주도권 싸움 게임 룰**로 바꾼 barge-in 트리비아입니다.
Cloud Run에서 운영되고 Firestore에 점수가 저장됩니다.”

화면: 아키텍처 다이어그램 5초 오버레이 + 최종 점수

---

## 촬영 팁(필수)
- barge-in은 **반드시 2회** 보여주기 (재현성)
- ‘끊는 순간’에 오디오가 확 끊기게(체감) 녹음 레벨 조절
- Firestore 저장 화면은 3~5초면 충분(증빙용)
